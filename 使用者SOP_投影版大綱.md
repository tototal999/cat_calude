# ClaudeCat 使用者 SOP（投影版）

## 設計原則

- 16:9 投影；標題至少 36pt、正文至少 18pt、頁尾至少 11pt。
- 每頁只傳達一個操作重點；以截圖、卡片與步驟取代密集文字。
- 不在簡報中揭露公司端點、模型名稱或憑證。

## 1. 封面

**ClaudeCat 使用者 SOP**

副標：聊天、文件問答與安全結束，三分鐘即可上手。

## 2. ClaudeCat 能做什麼

**桌面上的公司 AI 助手：可直接提問，也可依文件來源回答。**

多數同事無法直接上外網使用 LLM；ClaudeCat 將公司已提供的內網 AI 整合到桌面貓咪中。使用者不必另外開啟網頁、命令列、Ollama 或模型程式，即可開始提問、翻譯與閱讀文件，明顯降低使用 AI 的門檻。

| 功能 | 可以做什麼 | 使用方式 |
|---|---|---|
| 快速提問 | 整理工作重點、解釋錯誤訊息、協助撰寫或翻譯內容 | 點一下桌面貓咪，輸入問題後按 Enter |
| 長回答卡片 | 顯示較完整的分析、建議或 SQL／程式碼說明，可複製與追問 | 問題需要較長答案時自動展開 |
| 切換到 LLM 介面 | 開啟完整交談介面，進行多輪長對話、查看對話紀錄、使用 JSON／翻譯等工具 | 右鍵點貓咪，選「交談（LLM 介面）…」 |
| 對話紀錄與匯出 | 保留過去對話，可回看、繼續對話、匯出 Markdown；簡報大綱可匯出 PPTX | 在 LLM 交談介面的側欄與操作列使用 |
| 附件分析 | 將可讀取的文字或 Excel 附件加入目前問題的上下文 | 在 LLM 交談介面選擇附件後送出 |
| 分析文件 | 匯入 PDF、Word、PowerPoint、Excel 等文件，先建立本機索引，再產生摘要、流程／SOP、表格整理或文件比較 | 右鍵點貓咪，選「文件助手…」 |
| 文件問答 | 依分析後的文件內容提問；回答只能使用找到的證據 | 在文件助手輸入問題 |
| 文件會議包 | 一鍵完成「摘要 → 會議重點 →（可選）英文翻譯 → Markdown 成果」，過程可看步驟、可取消、失敗可重試 | 在文件助手按「建立文件會議包」 |
| 文件來源引用 | 回答會附頁碼、段落、投影片或儲存格範圍；沒有證據時明確說明 | 在文件助手結果下方查看來源 |
| JSON 工具 | 格式化、壓縮、驗證 JSON，顯示錯誤行／欄與 JSONPath | 右鍵點貓咪，選「JSON 工具…」 |
| 翻譯工具 | 來源語言可自動偵測或指定英文／繁中／簡中，目標語言為繁中／簡中／英文，並可雙箭頭互換；支援一般、技術、商務、中英對照並保護程式碼與識別碼 | 右鍵點貓咪，選「翻譯…」 |
| 模型設定 | 檢視或調整公司 LLM 連線、預設模型與任務路由 | 右鍵點貓咪，選「模型設定…」 |
| 切換 Skin | 依個人偏好切換藍貓、乳牛貓或布偶貓外觀 | 右鍵點貓咪，在「切換 Skin」子選單選擇外觀 |
| 桌寵與系統匣管理 | 拖曳移動貓咪；可從系統匣顯示／隱藏、快速提問、開啟文件助手或結束 | 左鍵拖曳；或使用工作列右下角的系統匣圖示 |
| 排程提醒 | 建立每日、每週或每小時提醒 | 右鍵點貓咪，選「排程…」 |
| 用量顯示（選用） | 顯示本機 Claude／Codex 的用量狀態 | 在設定中個別開關；不影響聊天與文件功能 |

**使用邊界：**聊天與文件回答使用公司設定的內網 LLM；Claude／Codex 用量功能完全獨立，預設關閉。使用者不需要外網 LLM 帳號，也不需要自行設定模型。

## 3. 目前使用的模型

**使用公司設定的內網模型，使用者不需選擇。**

- 這是公司設定的內網 OpenAI-compatible 模型；使用者不需要外網帳號、API Key，也不需要自行挑選模型。
- 目前聊天、文件問答與翻譯都使用公司統一的預設模型。
- 模型以直接、快速產出回答為主；需要依文件確認時，ClaudeCat 仍會先檢索本機文件證據並附上來源。
- JSON 的格式化、驗證、搜尋與 JSONPath 是本機確定性工具，完全不呼叫模型。
- 本機 llama.cpp sidecar 目前未啟用；Claude／Codex 用量僅是可選顯示，聊天不會呼叫 Claude 或 Codex。

> 簡報不列出內網網址、模型名稱、API Key 或其他連線憑證。

## 4. Agenda

1. 快速提問與長回答
2. 切換到 LLM 交談介面
3. 分析文件、文件問答與來源引用
4. 文件會議包：一鍵產出可交付的 Markdown
5. 桌寵管理、Skin、排程與安全結束

## 5. 快速提問

**一般問題，不必另開聊天視窗**

1. 點一下桌面貓咪。
2. 在旁邊輸入問題後按 Enter。
3. 短答案直接顯示成氣泡。

範例：今天的工作重點怎麼整理？

## 6. 長回答

**需要細節時，展開成貓咪旁的卡片**

- 可複製答案。
- 可繼續追問。
- 完成後可收合。

範例：SQL Review、文件重點整理。

## 7. 切換到 LLM 交談介面

**需要多輪對話或工具箱時，開啟完整 LLM 介面**

1. 右鍵點選桌面貓咪。
2. 選「交談（LLM 介面）…」。
3. 在完整介面進行多輪對話，或切換 JSON／翻譯工具。

快速問題仍建議直接點貓咪，較不打斷工作流程。

**此介面也可使用：**

- 側欄的對話紀錄：回看或繼續之前的對話。
- 附件：將文字或 Excel 檔案加入本次問題。
- Markdown／程式碼複製：複製回答中的內容。
- 簡報匯出：將已整理的簡報大綱匯出為 PPTX。
- `/` 快速提示詞：快速帶入常用提問。

## 8. 分析文件與文件問答

**把文件交給貓咪，再依來源提問**

1. 右鍵點貓咪，選「文件助手…」。
2. 選擇 PDF、Word、PowerPoint 或 Excel 文件，等待本機索引與分析完成。
3. 可執行摘要、流程／SOP、表格整理或比較文件。
4. 提問並查看答案下方的來源引用。

建議問題：誰需要批准？這份文件在說什麼？有哪些注意事項？

提醒：文件未提及時，系統會明確表示無法依文件確認。

## 9. 文件會議包

**一鍵把文件變成可交付的會議 Markdown**

1. 先完成第 8 頁的文件分析。
2. 需要英文版時，勾選「加入英文翻譯」。
3. 按「建立文件會議包」，依序執行：`retrieve_evidence`（找出證據）→ `summarize`（摘要）
   → `meeting_notes`（會議重點）→（可選）`translate` → `export_markdown`。
4. 畫面即時顯示每一步狀態：`✓` 完成、`●` 執行中、`○` 待執行、`✕` 失敗。

- 執行中可按「取消」；失敗或取消後可按「重新執行」（同一次失敗最多重試 3 次）。
- 中途失敗不會丟掉已完成的部分，會保留為「部分成果」。
- 結果會標示抽樣涵蓋範圍與每筆來源定位，並輸出 Markdown 檔路徑。
- 若上次程式被強制關閉，未完成的工作會標記為失敗並附說明，可直接重新執行。

## 10. 清理 Workflow 歷史

**成果會累積在本機，可自行清理**

1. 在文件助手上方按「清理 Workflow 歷史」。
2. 確認提示後，清除已結束的 Run 與 Markdown 成果。
3. 執行中的工作會保留，不會被清掉。

系統另會自動保留最近的執行紀錄，不需每次手動清理。

## 11. 工具箱：JSON 與翻譯

**不需要 LLM 的 JSON 工具**

- Format／Minify／Validate／搜尋／JSONPath。
- JSON 格式錯誤會顯示行與欄。

**需要語意處理的翻譯工具**

- 來源語言可自動偵測，或指定英文／繁中／簡中；目標語言為繁中／簡中／英文，可用 ⇄ 互換。
- 模式：一般／技術／商務／中英對照。
- 程式碼、SQL、JSON Key、API path 與檔名會保留原樣。

## 12. 切換 Skin

**可依個人偏好更換貓咪外觀**

1. 右鍵點選桌面貓咪。
2. 開啟「切換 Skin」子選單。
3. 選擇 bluecat、cowcat 或 ragdollcat。

切換後立即生效，重新啟動後也會保留最後一次選擇。

## 13. 用量顯示（選用）

**Claude／Codex 用量可個別開關**

- 不影響聊天或文件問答。
- 未安裝或未登入時顯示 `No use`。

## 14. 桌寵管理與排程

**維持桌面工作習慣，不必一直開著聊天視窗**

- 左鍵拖曳貓咪或用量徽章，可移到適合的位置；位置會保留。
- 系統匣可顯示／隱藏桌寵、快速提問、開啟文件助手與結束程式。
- 右鍵選「排程…」，可新增每日、每週或每小時提醒。

## 15. 安全結束與常見提示

**從系統匣安全關閉**

1. 在工作列右下角找到 ClaudeCat 圖示。
2. 右鍵選「結束」。
3. 確認貓咪已從桌面消失。

常見提示：缺少模型／服務請聯絡 IT；掃描型 PDF 需要 OCR；文件無資料時請改問文件確實出現的內容。

## 結尾訊息
